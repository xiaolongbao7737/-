## 二、写给"一、项目介绍"的内容

### 2.1 项目名称
**AI 手势数字识别平台**（AI Finger Digit Recognition Platform）

### 2.2 项目目标
构建一个 Web 端的实时手势数字识别系统，用户上传手势图片（数字 0-5），AI 模型自动识别并返回结果，同时提供数据采集、模型训练、数据分析等完整功能。

### 2.3 前端核心功能（一句话版本）
> 基于 HTML5/CSS3/原生 JavaScript 构建的 SPA 单页应用，包含 Dashboard、手势识别、数据分析、数据采集与训练四大模块，采用 Apple/OpenAI 白色极简设计风格。

### 3.3 前端架构

```
手势识别前端 (SPA)
├── Hash 路由系统（#dashboard / #recognize / #analytics）
├── 导航栏（sticky 顶部，三页面切换）
├── 页面 1: Dashboard（仪表盘）
│   ├── Hero 大标题区域
│   ├── 4 统计卡片（数据集/准确率/识别次数/训练次数）
│   ├── 系统信息面板（API 拉取）
│   └── AI 工作流程图（4 步骤）
├── 页面 2: 识别中心（核心功能）
│   ├── 左列 340px：上传区 + 示例图片按钮
│   ├── 中列 1fr：图片预览区
│   └── 右列 340px：识别结果 + 置信度圆环 + Top10 概率条 + 操作按钮
├── 页面 3: 数据分析（图表展示）
│   ├── Chart.js 饼图（数据集类别分布）
│   ├── Chart.js 柱状图（准确率对比）
│   ├── Chart.js 折线图（今日识别统计）
│   └── 训练历史 + 模型信息面板
└── 公共组件
    ├── Toast 消息提示（右上角滑入）
    ├── Modal 确认对话框
    └── 响应式断点（≤1150px 单列 / ≤768px 紧凑）
```

### 3.4 核心识别流程（关键交互逻辑）

```
用户选择图片（点击上传 / 按钮选择 / 拖拽上传）
       │
       ▼
前端 FileReader 读取 → 生成缩略图预览（图片淡入动画 imgIn 0.45s）
       │
       ▼
FormData 封装 → Fetch POST /predict → 后端返回 JSON
       │
       ▼
前端解析结果：
  ├── 识别数字（数字弹出动画 numPop 0.5s）
  ├── 置信度圆环（SVG stroke-dashoffset 过渡 0.7s）
  ├── Top10 概率条（width 过渡 0.6s）
  └── 自动记录到识别历史（存 localStorage + 后端同步）
```

### 3.5 CSS 布局关键技术点

- **三列等高固定**：`grid-template-columns: 340px 1fr 340px` + 每个格子 `height: 540px`，`align-items: start` 顶部对齐
- **图片铺满预览框**：`position: absolute; width: 100%; height: 100%; object-fit: contain` 完全填充
- **响应式断点**：`≤1150px` 三列变单列；`≤768px` 统计卡片 4 列变 2 列、导航紧凑
- **动画**：6 种 CSS 动画（图片淡入、数字弹出、圆环进度、概率条、按钮 hover、上传区 hover）

### 3.6 调用的后端 API（共 17 个，前端涉及以下）

| API | 用途 | 前端调用场景 |
|:----|:-----|:-----------|
| `POST /predict` | 上传图片识别 | 识别页面核心调用 |
| `GET /api/system/info` | 系统信息 | Dashboard 加载 |
| `GET /api/history` | 识别历史 | 识别页面历史列表 |
| `POST /api/history/save` | 保存记录 | 每次识别后自动保存 |
| `POST /api/history/clear` | 清空历史 | Modal 确认后调用 |
| `GET /api/sample/<digit>` | 训练集示例 | 示例图片按钮加载 |
| `GET /api/dataset/quality` | 数据质量统计 | 数据分析页面 |
| `GET /api/train/progress` | 训练进度轮询 | 采集页面每 2 秒轮询 |
| `GET /api/training/history` | 训练历史 | 数据分析页面 |
| `GET /api/models/versions` | 模型版本 | 数据分析页面 |
| `GET /api/capture/stats` | 采集统计 | 采集页面 |

### 3.7 数据采集与训练页面的交互

- 6 个数字卡片（0-5），点击选中目标数字
- 文件上传（支持多选），预览区显示待上传图片
- 采集统计面板，实时显示每个数字已采集数量
- **训练进度可视化**：进度条 + Epoch / 准确率 / Loss / 模式 / 学习率 / 耗时，每 2 秒 Ajax 轮询
